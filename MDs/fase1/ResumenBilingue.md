# Resumen del proyecto

## Resumen 

Analizando ideas para nuestro proyecto nos dimos cuenta de que los usuarios necesitan muchas aplicaciones y sitios web para realizar un seguimiento del contenido multimedia que consumen, lo cual resulta molesto. Debido a esto, hemos decidido crear MediaLibrary, una solución para centralizar todas estas plataformas en una sola.

Para MediaLibrary, hemos establecido algunos objetivos generales que queremos conseguir en nuestro desarrollo:   
Queremos que los usuarios puedan realizar un seguimiento de sus películas, libros y videojuegos. Puntuar dichos contenidos multimedia, crear una lista de "planeado ver" y añadir una función para que los usuarios descubran nuevo contenido. También queremos que se pueda crear un perfil que otros usuarios puedan ver y seguir y que en dichos perfiles se puedan ver logros desbloqueados dependiendo de los contenidos de sus librerías.

Para llevar a cabo este proyecto, también hemos creado una serie de objetivos específicos más técnicos relacionados con la arquitectura del software:   
Queremos crear una base de datos en PostgreSQL para almacenar los datos de los usuarios y el contenido multimedia de la aplicación. También planeamos crear una API en Java Spring Boot para gestionar la base de datos, las peticiones de los clientes y hacer llamadas a las APIs externas para añadir nueva información. Además, queremos construir un cliente web (en React) y una aplicación móvil (en Kotlin) para los usuarios. Por último, queremos alojar nuestro proyecto utilizando los servicios de AWS (Amazon Web Service).

## Summary

While analyzing ideas for our project, we realized that users need multiple apps and websites to keep track of the media they consume, which can be annoying. Because of this, we decided to create MediaLibrary, a solution to centralize all these platforms into a single platform.

For MediaLibrary, we've established some general goals we want to achieve in our development:

We want users to be able to keep track of their movies, books, and video games. We want to rate these media content, create a "planned to watch" list, and add a feature for users to discover new content. We also want to be able to create a profile that other users can view and follow, and on these profiles, we want to display achievements unlocked based on the content in their libraries.

To carry out this project, we've also created several more technical specific goals related to the software architecture:
We want to create a PostgreSQL database to store user data and the application's media content. We also plan to create a Java Spring Boot API to manage the database, client requests, and make calls to external APIs to add new information. We also want to build a web client (in React) and a mobile app (in Kotlin) for users. Finally, we want to host our project using AWS (Amazon Web Service).
